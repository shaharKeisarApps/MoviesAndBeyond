# Release v0.0.2 — UI Polish & Landscape Support

**Priority:** Critical
**Target:** v0.0.2
**Status:** In Progress

---

## Scope

### 1. Landscape Mode Support (All Screens)

**Requirement:** Remove portrait lock and add responsive adaptive layouts to every screen.

#### Implementation Plan

**Step 1: Remove portrait lock**
- File: `app/src/main/AndroidManifest.xml`
- Remove: `android:screenOrientation="portrait"` from MainActivity

**Step 2: Add WindowSizeClass dependency**
- Add `material3-window-size-class` dependency
- Calculate `WindowSizeClass` in `MainActivity.kt`
- Pass through `CompositionLocalProvider` or function params

**Step 3: Adapt each screen for landscape**

| Screen | Portrait Layout | Landscape Layout | Key Changes |
|--------|----------------|------------------|-------------|
| Movies FeedScreen | Single-column lazy list with horizontal carousels | Side-by-side category panels or wider grid | Increase grid columns based on width |
| TV FeedScreen | Same as Movies | Same as Movies | Mirror Movies approach |
| Search Screen | Fixed grid columns | Adaptive grid with more columns | `GridCells.Adaptive(minSize)` instead of `Fixed` |
| Details Screen | Vertical: backdrop → info → cast | Horizontal: backdrop left, info+cast right | Two-pane layout for Expanded width |
| Credits Screen | Single-column list with sticky headers | Two-column or wider cards | Adaptive grid |
| Auth Screen | Centered card | Wider max-width constraint | `widthIn(max = 480.dp)` |
| You Screen | Single-column profile | Side panel for profile, main area for library | Two-pane for wide |
| Library Items Screen | Grid listing | Wider adaptive grid | More columns |
| Onboarding Screen | Full-screen vertical | Constrained width with side illustration | Max-width + illustration |

**Step 4: Floating Navigation Bar landscape adaptation**
- In landscape: Consider NavigationRail instead of bottom bar (Material 3 guidance)
- Or: Reduce bar height, adjust padding for landscape
- File: `core/ui/FloatingNavigationBar.kt`

**Step 5: HazeScaffold / NestedScaffold updates**
- Ensure `safeDrawing` insets work correctly in landscape
- Handle horizontal system bars (cutout/notch areas)

### 2. Edge-to-Edge & Navigation Bar Fixes

**Problem 1: Content behind system navigation bar**

Multiple screens set `contentWindowInsets = WindowInsets(0, 0, 0, 0)` which removes all inset protection, causing content to render behind the gesture navigation bar.

**Affected screens:**
- `feature/movies/FeedScreen.kt` — `WindowInsets(0, 0, 0, 0)`
- `feature/tv/FeedScreen.kt` — `WindowInsets(0, 0, 0, 0)`
- `feature/search/SearchScreen.kt` — `WindowInsets(0, 0, 0, 0)`
- `feature/auth/AuthScreen.kt` — `WindowInsets(0, 0, 0, 0)`
- `feature/you/YouScreen.kt` — `WindowInsets(0, 0, 0, 0)`
- `feature/details/CreditsScreen.kt` — `WindowInsets(0, 0, 0, 0)`

**IMPORTANT — Read `.claude/skills/device-validate/SKILL.md` 'Edge-to-Edge Inset Chain' first!**

The `WindowInsets(0, 0, 0, 0)` on inner scaffolds is **INTENTIONAL** — the outer `HazeScaffold` with `contentWindowInsets = WindowInsets.safeDrawing` handles system insets. The actual bug is the floating bottom bar overlaying content without pushing it up.

**Fix approach:**
- Add bottom content padding (~96dp = 80dp bar + 8dp×2 padding) to all scrollable lists so last items clear the floating nav bar
- Do NOT change `contentWindowInsets = WindowInsets(0, 0, 0, 0)` on inner scaffolds — this prevents double-consuming
- Do NOT add `navigationBarsPadding()` to inner content — HazeScaffold already handles system nav bar

**Problem 2: Details screen — content behind floating nav bar at bottom**

The details screen's scrollable content extends behind the floating navigation bar. The backdrop intentionally goes behind the status bar (cinematic effect — see device-validate SKILL.md) but the bottom content needs padding to clear the floating bar.

**Fix:** Add bottom spacing to the last item in the scrollable content equal to the floating nav bar height.

**Problem 3: Credits/Cast screen — edge-to-edge problem**

`CreditsScreen.kt` has no bottom padding, so the last cast member cards get hidden behind the floating nav bar.

**Fix:** Add a bottom content padding or spacer to the LazyColumn equal to the floating nav bar height (~96dp).

- **Phase 1**: Code analysis (M3 compliance, Compose patterns, accessibility, inset architecture)
